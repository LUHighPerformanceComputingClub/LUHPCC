# Pi Cluster SLURM Guide
#### Written By Ben Peterson + AI

This guide walks through setting up a Raspberry Pi cluster using SLURM for workload management.

---

## Prerequisites
- At least two Raspberry Pi systems (1 master, 1+ compute nodes)
- Ethernet switch/network connectivity
- Power and Ethernet cables
- Raspberry Pi OS installed on all nodes
- SSH access between nodes
- Internet access for package installation

---

## System Updates

Run on ALL nodes:
```bash
sudo apt update
sudo apt upgrade -y
```

---

## Install Required Packages

### On MASTER node:
```bash
sudo apt install munge build-essential libmunge-dev libssl-dev libpam0g-dev default-libmysqlclient-dev wget slurmctld slurmdbd mariadb-server -y
```

### On COMPUTE nodes:
```bash
sudo apt install munge slurmd -y
```

---


## Network Configuration (IMPORTANT)

### Edit `/etc/hosts` on ALL nodes
This is critical for SLURM communication.

```bash
sudo nano /etc/hosts
```

Example:
```
127.0.0.1       localhost
127.0.1.1       nameOfNode (master, slave)

# Cluster nodes
192.168.X.X     master
192.168.X.X     slave1
```

> Ensure all nodes can ping each other by hostname. Make sure that you edit this on each node.

---



## Configure Munge (Authentication)

### On MASTER:
```bash
sudo rm /etc/munge/munge.key
mungekey
sudo chown munge:munge /etc/munge/munge.key
sudo chmod 400 /etc/munge/munge.key
sudo systemctl enable --now munge
```

Copy key to compute nodes:
```bash
sudo scp /etc/munge/munge.key pi1@slave1:~/
```

---

### On COMPUTE node(s):
```bash
sudo cp munge.key /etc/munge/
sudo chown munge:munge /etc/munge/munge.key
sudo chmod 400 /etc/munge/munge.key
sudo systemctl enable --now munge
```

Test munge:
```bash
munge -n | unmunge
```

Expected output:
```
STATUS: Success (0)
```

## Distribute SLURM Config

Copy config to compute nodes:
```bash
sudo scp /etc/slurm/slurm.conf pi1@slave1:/etc/slurm/
```

---

## Create Required Directories

Run on ALL nodes:
```bash
sudo mkdir -p /var/spool/slurmctld
sudo mkdir -p /var/spool/slurmd
sudo chown -R slurm:slurm /var/spool/slurm*
```

### MASTER:
```bash
sudo systemctl enable --now slurmctld
```

### COMPUTE:

## Common Issues / Fixes

### Nodes stuck in UNKNOWN
```bash
scontrol update NodeName=slave1 State=RESUME
```

### Munge errors
- Ensure same key on all nodes
- Check permissions: `400`
- Check ownership: `munge:munge`
- Restart munge

### Hostname issues
- Verify `/etc/hosts` consistency across ALL nodes

### User issues
- When users are not unified, add a unified user with the below commands
```bash
sudo groupadd -g 1050 alice
sudo useradd -u 1050 -g 1050 -m -s /bin/bash alice
```
